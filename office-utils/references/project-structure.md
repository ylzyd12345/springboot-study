# 项目结构规范

## 标准目录结构

```
工作项目/
├── *.xlsx, *.xls, *.docx, *.pdf    # 原始数据文件（根目录）
├── scripts/                         # 处理脚本
│   └── {任务名}.py                  # 数据处理代码
├── 处理结果/                        # 输出目录
│   ├── {日期}_{任务}_工作记录.md    # 工作日志
│   └── {日期}_{任务}_{版本}.xlsx    # 处理后的文件
└── tools/                           # 工具库（可选）
```

## 命名规范

### 工作日志
```
{日期}_{任务简称}_工作记录.md
```
示例：`20251226_死亡核查_工作记录.md`

### 处理结果文件
```
{日期}_{任务}_{版本/类型}.{ext}
```
示例：
- `20251205_追溯码核查_分析报告.xlsx`
- `20251212_死亡核查_初始版.xlsx`
- `20251215_死亡核查_校对版.xlsx`

### 处理脚本
```
{动作}_{对象}.py
```
示例：`fill_province_table.py`、`compare_versions.py`

## 工作流程

1. **放入文件** - 将原始数据文件放入项目根目录
2. **AI 分析** - 使用 Skill 分析文件，生成工作日志
3. **处理数据** - 编写脚本处理数据，脚本放 `scripts/`
4. **输出结果** - 处理结果放 `处理结果/`，更新工作日志
5. **版本迭代** - 收到反馈后，新版本按命名规范保存，记录变化
